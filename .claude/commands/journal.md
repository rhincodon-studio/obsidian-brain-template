# /journal — 建立工作日誌

建立或開啟今日的 Daily Note，填入當日摘要。

## 執行步驟

1. **確認今日日誌**
   - 目標路徑：`journals/YYYY-MM-DD.md`（以當前日期）
   - 若已存在 → 直接讀取並顯示現有內容，詢問要補充什麼
   - 若不存在 → 用模板建立新檔案

2. **建立新日誌（不存在時）**
   套用 `System/templates/daily.md` 模板，加上 frontmatter：
   ```yaml
   ---
   type: journal
   date: YYYY-MM-DD
   ---

   # YYYY-MM-DD
   ```
   然後接模板內容。

3. **引導填寫**
   依序詢問（使用者可跳過任何一項）：
   - **今天做了什麼？** → 填入「碎片」
   - **有沒有未完成的事？** → 填入「任務」
   - **有什麼值得記下來的想法？** → 填入「洞察」

4. **關聯檢查**
   - 若使用者提到某個專案 → 主動關聯 `Projects/` 下的對應檔案
   - 若提到某個概念 → 詢問是否要寫入 `Resources/`

5. **完成回報**
   ```
   日誌已建立：journals/YYYY-MM-DD.md
   本週已記錄：X 天（執行 /reflect 可產生週報）
   ```

## 設計原則

- **低摩擦**：不強迫填完所有欄位，有記錄比沒記錄好
- **主動關聯**：對話中識別到可沉澱的內容，主動詢問是否歸檔

## 注意事項

- 日誌檔案已在 `.gitignore` 排除，不會進入 git
- 若要回顧過去某天，直接說「看 2024-01-15 的日誌」即可
