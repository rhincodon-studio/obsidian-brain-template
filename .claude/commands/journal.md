# /journal — 建立工作日誌

建立或開啟今日的 Daily Note，填入當日摘要。

## 執行步驟

1. **確認今日日誌**
   - 目標路徑：`journals/YYYY-MM-DD.md`（以當前日期）
   - 若已存在 → 直接讀取並顯示現有內容，詢問要補充什麼
   - 若不存在 → 用模板建立新檔案

2. **建立新日誌（不存在時）**
   套用 `System/templates/daily.md` 模板，加上 frontmatter：
   ```yaml
   ---
   type: journal
   date: YYYY-MM-DD
   ---

   # YYYY-MM-DD
   ```
   然後接模板內容。

3. **引導填寫**
   依序詢問（使用者可跳過任何一項）：
   - **今天做了什麼？** → 填入「碎片」
   - **有沒有未完成的事？** → 填入「任務」
   - **有什麼值得記下來的想法？** → 填入「洞察」

4. **沉澱到 Inbox**
   日誌填寫完成後，掃描內容，將符合以下條件的項目寫入 `Inbox/YYYY-MM-DD-journal.md`：
   - 洞察欄位有內容 → 整段寫入
   - 碎片中出現「我覺得」「發現」「想到」等主觀判斷 → 摘錄
   - 任務中有未完成且跨日的項目 → 標記為待處理

   Inbox 檔案格式：
   ```markdown
   ---
   source: journal
   date: YYYY-MM-DD
   ---

   # 來自 YYYY-MM-DD 日誌的待整理素材

   [摘錄的洞察與碎片]
   ```

   若日誌內容全是流水帳、無可沉澱的洞察 → 跳過此步驟，不建立空檔案。

5. **關聯檢查**
   - 若使用者提到某個專案 → 主動關聯 `Projects/` 下的對應檔案

6. **完成回報**
   ```
   日誌已建立：journals/YYYY-MM-DD.md
   已寫入 Inbox：Inbox/YYYY-MM-DD-journal.md（X 條素材待整理）
   本週已記錄：X 天（執行 /reflect 可產生週報）
   ```

## 設計原則

- **低摩擦**：不強迫填完所有欄位，有記錄比沒記錄好
- **主動關聯**：對話中識別到可沉澱的內容，主動詢問是否歸檔

## 注意事項

- 日誌檔案已在 `.gitignore` 排除，不會進入 git
- 若要回顧過去某天，直接說「看 2024-01-15 的日誌」即可
