# /writing — 啟動寫作模式

載入使用者的寫作風格規則，按個人風格執行寫作任務。

## 使用方式

```
/writing [主題或任務描述]
```

## 執行步驟

1. **載入寫作情境**

   依序讀取：
   - `Identity/PROFILE.md` — 個人偏好（溝通風格、禁忌）
   - `System/OPERATING_RULES.md` — 操作規則（格式要求）
   - `Resources/` 下的寫作相關方法論
   - 相關的歷史案例（若有）

2. **確認寫作任務**
   ```
   ## 寫作任務確認

   **主題：** [使用者輸入的主題]
   **類型：** [文章/郵件/報告/提案/...]
   **目標讀者：** [推測或詢問]
   **預期長度：** [推測或詢問]

   已載入的風格規則：
   - [從 PROFILE 載入的偏好]
   - [從 OPERATING_RULES 載入的規則]

   是否正確？或需要調整？
   ```

3. **產出大綱**
   - 先產出結構大綱
   - 等待使用者確認或調整
   - 確認後再展開內容

4. **執行寫作**
   - 按照載入的風格規則寫作
   - 使用使用者慣用的詞彙和句式
   - 避免使用者標記為禁忌的表達方式

5. **輸出結果**
   - 產出完整內容
   - 標記可能需要調整的部分
   - 提供修改建議

6. **學習回饋**
   - 若使用者修改了內容
   - 觀察修改模式
   - 靜默寫入候選池（不打斷）

## 設計原則

- **提示詞驅動**：風格規則在提示詞，模型升級自動變強
- **一魚多吃**：寫作過程同時學習偏好

## 風格規則格式

在 `Identity/PROFILE.md` 中可定義：

```markdown
## 寫作風格

- 偏好短句，避免冗長從句
- 使用主動語態
- 技術文章可用英文術語，但解釋用中文
- 避免使用「其實」「基本上」等填充詞

## 禁忌

- 不使用表情符號（除非明確要求）
- 不使用「親愛的」開頭
```

## 注意事項

- 第一次使用時，若 PROFILE 無寫作偏好，詢問使用者
- 觀察到的新偏好寫入候選池，不直接生效
- 重要文件（合約、正式信函）建議人工最終審核
