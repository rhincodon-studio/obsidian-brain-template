# /maintain — 全庫健康檢查

掃描整個知識庫，找出問題並輸出診斷報告。

## 執行步驟

1. **統計概覽**
   - 計算各 PARA 目錄的檔案數量
   - 計算總篇數

2. **檢測知識壞味道**

   ### 孤島檢測
   - 找出沒有任何 `[[連結]]` 指向的檔案
   - 排除 `_index.md` 和 `journals/`

   ### 冗餘檢測
   - 找出 summary 高度相似的檔案
   - 標記可能重複的內容

   ### 僵化檢測
   - 找出 `status: active` 但超過 90 天未更新的檔案

   ### 混沌檢測
   - 計算 Inbox 堆積數量
   - 若 > 10 篇，標記為警告

   ### 斷裂檢測
   - 找出引用了不存在檔案的 `[[連結]]`

   ### 空洞檢測
   - 找出 `_index.md` 成員清單為空的目錄

3. **Frontmatter 一致性檢查**
   - 找出缺少必要欄位的檔案（para, summary）
   - 找出 `para` 值與實際路徑不符的檔案

4. **說明檔完整性檢查**
   - 找出有檔案但沒有 `_index.md` 的目錄
   - 找出 `_index.md` 成員清單與實際檔案不符的情況

5. **輸出診斷報告**
   ```
   ## 知識庫健康報告

   📊 **統計**
   - 總篇數：X 篇
   - Projects: A | Areas: B | Resources: C | Archives: D

   🔴 **嚴重問題**（需立即處理）
   - [問題描述]

   🟡 **警告**（建議處理）
   - [問題描述]

   🟢 **建議**（可選處理）
   - [問題描述]

   ---
   上次維護：YYYY-MM-DD
   本次維護：YYYY-MM-DD
   ```

6. **提供修復選項**
   - 對每個問題，提供一鍵修復選項
   - 使用者確認後執行修復

## 設計原則

- **自我維護**：系統自己知道什麼時候需要維護
- **使用者是審核者**：只展示問題和方案，等使用者確認

## 注意事項

- 不自動修復，只報告和建議
- 嚴重問題優先展示
- 記錄維護時間到 `System/tasks.md`
