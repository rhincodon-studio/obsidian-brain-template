# /reflect — 週度覆盤

聚合過去 7 天的 journal，提煉關鍵洞察，識別重複模式。

## 執行步驟

1. **聚合週記錄**
   - 讀取 `journals/` 下最近 7 天的日記
   - 若不足 7 天，讀取所有可用的日記
   - 提取每日的碎片、任務、洞察

2. **識別模式**
   ```
   ## 行為模式
   - 重複出現的主題
   - 時間分配傾向
   - 情緒波動規律

   ## 任務執行
   - 完成率
   - 拖延模式
   - 高效時段
   ```

3. **提煉洞察**
   - 哪些事情做得好？
   - 哪些事情可以改進？
   - 有什麼新發現？

4. **更新候選池**
   - 識別到的偏好 → 寫入 `System/candidates.md`
   - 🔴 跨多日重複出現
   - 🟡 與當前專案相關
   - 🟢 單次觀察

5. **產出週報**
   ```
   ## 週度覆盤 | YYYY-MM-DD ~ YYYY-MM-DD

   ### 本週關鍵
   - [3-5 個最重要的事項]

   ### 完成事項
   - [x] 完成的任務列表

   ### 持續進行
   - [ ] 跨週的任務

   ### 洞察與反思
   - [本週學到的東西]

   ### 下週焦點
   - [下週最重要的 1-3 件事]
   ```

6. **存檔週報**
   - 寫入 `journals/YYYY-Wxx.md`（週報格式）

## 設計原則

- **一魚多吃**：碎片自動聚合成洞察
- **自我進化**：識別偏好，更新候選池

## 注意事項

- 若本週無日記，提醒使用者建立記錄習慣
- 週報不取代日記，而是聚合與提煉
- 偏好候選不自動生效，等待 `/digest` 確認
